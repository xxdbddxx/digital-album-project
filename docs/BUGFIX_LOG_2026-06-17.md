# 2026-06-17 更新与 Bug 修复日志

对应提交：`c5fc83e Integrate smart album backend and portrait display fixes`

## 一、后端与网页端

- 新增 Flask 后端服务，包含照片库、图片上传、物理设备台、智能助手、图库管理等页面。
- 新增网页端设备照片库管理能力，支持查看设备图库、移出设备库、同步控制 ESP32 当前显示内容。
- 新增物理设备台控制项：上一张、下一张、当前照片 ID、横屏/竖屏切换、立即强制切换照片。
- 上传图片时“悄悄话”不再强制必填；用户填写时使用用户文案，未填写时可调用 Gemini 多模态模型生成图片描述。
- 新增 Gemini API 图片理解与中文短文案生成逻辑。
- 新增设备端 RGB565 照片接口，支持同一照片按横屏/竖屏生成不同显示变体。
- 新增 `.gitignore` 规则，排除数据库、上传缓存、输出缓存、TTS 模型、mp3 音频等本地运行文件，避免仓库过大。

## 二、ESP32 设备端

- 合入 LVGL 主界面，包括顶部状态栏、底部文案栏、照片显示、图标资源、占位图。
- 新增 `photo_client`，支持从后端拉取照片列表、下载 RGB565 图片、上报设备心跳。
- 新增网页远程控制指令处理，支持切图、横竖屏切换、设备状态同步。
- 合入语音助手相关模块，包括 WebSocket 语音链路、音频缓冲、流式播放、语音任务框架。
- 外设代码整理为 `include/` 与 `src/` 结构，包括 AHT20、PCF8574、I2S、TCA9548A、香薰控制等模块。
- 保留当前硬件使用的 PCF8574 地址，并以本地修复后的音频播放逻辑为准。

## 三、显示与图片渲染修复

- 修复上传图片在屏幕上只显示上半部分、横向重复的问题。
- 修复设备内置图片横向重复、中间断线的问题。
- 修复 RGB565 图片尺寸与屏幕方向不匹配导致的显示异常。
- 修复竖屏模式下图片比例不合理的问题。
- 竖屏渲染改为自适应策略：
  - 接近竖屏比例的图片按竖屏铺满显示。
  - 横图优先完整显示，空白区域用模糊背景填充。
  - 横向画布但中间实际为竖图主体的图片，会先自动检测并提取中间有效主体，再按竖图显示。
- 修复旧 `portrait_cover` 缓存导致设备继续拿到旧裁剪图的问题。
