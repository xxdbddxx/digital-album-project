# 🧠 云端电子相册后端服务 (Backend) 部署与运行指南

本后端服务是云端电子相册的**“大脑与数据分发中心”**。它负责执行高阶多模态大模型（VLM）照片分析、无损 RGB565 裁剪与压缩、Flask Web 交互服务器提供，以及实时手机照片上传的高阶弹窗推送逻辑。

---

## 📂 照片的流动与放置位置

后端的照片数据在生命周期中分为**两大通道**：

### 1. 主相册通道（日常播发照片）
* **物理放置路径**：请将您的照片原图（JPG / PNG 等）放置在电脑或挂载 NAS 的物理目录中。
* **路径配置**：在 [config.py](file:///w:/Desktop/digital_album_project/backend/config.py) 的第 7 行进行配置，例如：
  ```python
  IMAGE_DIR = r"W:\Desktop\Data\图片\photo\new"
  ```

### 2. 手机/网页实时上传通道
* **自动存储路径**：当您通过手机或电脑访问 Web 页面上传照片时，原图会被自动接收并保存在 `backend/upload_photos` 目录下。
* **RGB565 转换路径**：上传的照片会被自动转换并保存在该目录下的专属子目录中，并在 `upload.db` 记录元数据。

## 📁 核心目录与架构解析

重构后的后端采用了极具专业度的模块化架构，彻底告别冗余，职责分明：

*   **`server.py`**：核心 Web 后端服务与路由总线，随时保持运行。
*   **`config.py`**：全局配置文件，统领全部核心参数。
*   **`templates/` & `static/`**：存放高阶 Glassmorphism 响应式前端交互页面（HTML/CSS/JS）。
*   **`tasks/`**：存放如 `analyze_photos.py`（AI 照片分析）与 `render_daily.py`（每日格式渲染）的后台长效计算任务。
*   **`tools/`**：存放如 IP 嗅探、默认图生成、专属字库提取等单次运行实用小工具。
*   **`services/`**：存放如 `voice_server.py` 的旁路独立辅助服务。
*   **`backup/`**：历史旧版代码及日志。

## 🚀 启动指引与运行流程

在部署好 Python 3.10+ 环境后，请按照以下步骤**依次执行**：

### 第一步：安装依赖库
在 `backend` 目录下打开终端，运行：
```bash
pip install -r requirements.txt
```

### 第二步：数据采集与 AI 智能打标入库 (双轨制)

相册后端支持两种不同的照片拉取和处理流程，您可以根据使用场景自由组合使用：

#### 方案 A：从本地硬盘拉取 (深度 AI 分析与打分)
如果您习惯手动整理高画质照片，请将原图拷入 `config.py` 中配置的 `IMAGE_DIR` 目录，然后手动运行：
```bash
python tasks/analyze_photos.py
```
* **完整过程与分析**：
  1. **扫描排重**：脚本会扫描本地目录，与 `photos.db` 对比并跳过已存在的记录。
  2. **深度提取**：全面解析照片的 EXIF 信息，并利用内建的 CSV 数据将 GPS 经纬度精准解析为具体的城市与地理位置。
  3. **云端精细推理**：调用云端高级大模型（如阿里百炼 `qwen-vl-max`），配合专业级系统提示词，生成极具情绪价值和文艺气息的“旁白短句”。
  4. **多维打分入库**：基于构图、色彩自动评估“美观分”，基于人物、情感评估“记忆分”，最后将丰富的元数据写入 `photos.db`。

#### 方案 B：从百度网盘自动同步 (轻量级 AI 快速归档)
如果您希望在外面用手机随手把照片远程传给相册，只需把照片上传至百度网盘的 `/apps/digital_album` 目录下，并运行（可结合系统定时任务或前端面板 `/api/baidu/sync` 接口触发）：
```bash
python tasks/baidu_sync_task.py
```
* **完整过程与分析**：
  1. **云端抓取**：通过 Baidu PCS API 抓取网盘专属目录下的新增照片，并获取云端真实下载直链 (`dlink`)。
  2. **下载归档**：将原图自动分块流式下载到本地 `IMAGE_DIR/baidu_cloud` 专属子目录中。
  3. **本地极速推断**：直接调用您本地部署的显卡大模型（如 Ollama `llava:7b`），用一句话和3个核心关键词快速“扫视”照片，无需消耗云端 API 额度。
  4. **默认打分入库**：提取基础尺寸与时间后，赋予默认的高分（记忆分85，审美分80）直接插入 `photos.db`，确保明早醒来即可看到新图。

> [!TIP]
> **流程交集说明**：百度同步脚本（方案 B）处理完的照片由于已录入数据库，若您事后再次运行深度分析脚本（方案 A），会触发排重机制被自动跳过。即相册贯彻的原则是：**百度传来的新图用本地卡极速跑以求快，本地珍藏的图库用云端大模型精修以求稳。**

### 第三步：照片无损缩放与 RGB565 转码
运行转码服务：
```bash
python tasks/render_daily.py
```
* **发生了什么**：根据今日日期，脚本从 `photos.db` 中回溯并选出高分照片（如大于 70 分），并对其执行：
  1. 比例智能裁剪：将其裁剪并缩放到实机屏幕要求的 **竖屏 480 × 800** 分辨率。
  2. 颜色无损转码：将其转码为 ESP32 硬件直接支持的 **16位 RGB565 二进制文件**（`.rgb565`），这能免除 ESP32 极其吃力的 JPEG 解压开销，实现瞬间闪显。
  3. 保存在 `backend/output` 目录下。

### 第四步：启动 Flask 主 Web 服务器
启动照片分发和手机上传服务：
```bash
python server.py
```
* **发生了什么**：启动服务后，会监听在端口 `8765`。它搭载了极致科技感与温情的 WebUI，包含四大核心面板：
   - 🌟 **`/review`**：高颜值玻璃拟态照片库，支持 AI 数据透视、多维度筛选与卡片就地旁白编辑。
   - 🚀 **`/upload`**：配备 3D CSS ESP32 手机屏幕真机模拟渲染器的惊喜传送门。
   - 🎛️ **`/device`**：无刷新轮询心跳的物理终端监控台，支持远端强切照片与 3 通道智能香薰操控。
   - 🤖 **`/dialogs`**：左侧与内置引擎进行情感对话，右侧垂直 Timeline 永久审计每次交互与状态联动。
   
### 常见问题与调试指南
* **仅需运行 `server.py`**：后端的其他脚本（如 `sync_lan_ip.py`、`render_daily.py`）是一次性离线工具，仅在需要更新 IP 或生成每日图片时手动执行。日常使用只启动 Flask 服务器即可。
* **设备卡在欢迎界面**：
  1. 检查 `sdkconfig.defaults` 中 `CONFIG_SERVER_URL` 是否与本机局域网 IP（`ipconfig` 可查看）完全一致。
  2. 重新执行 `idf.py reconfigure && idf.py build flash`，确保固件使用最新的服务器地址。
  3. 确认 ESP32 已成功连接 Wi‑Fi（查看串口日志中 `Got ip`、`WIFI_CONNECTED_BIT`）。
  4. 确认 PC 防火墙已放行 TCP 8765 端口。
* **常用调试命令**：
  - `curl http://<PC_IP>:8765/health`  检查后端是否存活。
  - `idf.py monitor` 观察 ESP32 启动日志。
  - `python sync_lan_ip.py`  自动同步 IP（可选）。
  - 上传网页：提供 `/upload`（手机上传界面）与 `/api/upload/check`（供实机 30 秒一次轮询是否有新上传照）。

### 第五步：启动语音交互服务器 (可选)
如果您希望使用相册的语音唤醒和交互（如“下一张照片”），请开启语音 WebSocket 服务器：
```bash
python services/voice_server.py
```
* **发生了什么**：开启在端口 `8888` 的 WebSocket 语音流式传输协议，负责将实机 MIC 采集的音频传输至 LLM 大模型处理，并控制相册做出相应反馈。

## ⚙️ 核心配置参数表

项目的核心配置由**后端配置文件**与**终端实机参数**共同维护，它们必须保持一致：

### 1. 后端 API 接口与大模型配置
请直接编辑 [backend/config.py](file:///w:/Desktop/digital_album_project/backend/config.py)
* `API_CHANNELS`：填入您的通义千问 API 密钥（`api_key`）。
* `DOWNLOAD_KEY`：默认是 `"yourdownloadkey"`，用以在传输中防盗链，可自定义。
* `FLASK_PORT`：默认监听端口 `8765`。
* `SURPRISE_UPLOAD_PASSWORD`：手机上传页面的密码，留空代表家庭公开直接上传。

### 2. ESP32 终端的 Wi-Fi 与 IP 配置 (免去修改 C 源码)
相册的实机配置完全使用 ESP-IDF 的 Kconfig 机制。您不需要在 C 代码里写死任何配置，只需在实机项目根目录下：
1. 打开终端，运行菜单配置命令：
   ```bash
   idf.py menuconfig
   ```
2. 进入 **`Network Configuration` (网络配置)** 菜单中：
   * **`Wi-Fi SSID`**：填入您家里的 Wi-Fi 名称。
   * **`Wi-Fi Password`**：填入 Wi-Fi 密码。
   * **`Photo Server URL`**：填入运行后端 Flask 服务器的 IP 及端口，格式为：
     ```text
     http://<您的电脑局域网IP>:8765
     ```
   * **`Download Key`**：填入与后端 [config.py](file:///w:/Desktop/digital_album_project/backend/config.py) 完全一致的密钥（默认：`yourdownloadkey`）。
3. 进入 **`Voice Assistant` (语音助手配置)** 菜单中：
   * **`LLM WebSocket Server URL`**：输入语音服务地址：
     ```text
     ws://<您的电脑局域网IP>:8888
     ```
4. 保存配置（按 `S` 键保存，回车确认），然后运行 `idf.py build flash` 编译烧录即可！

## 📱 网页端手机上传照片流程与实机动效衔接

1. **手机访问**：用手机浏览器访问 `http://<您的电脑局域网IP>:8765/upload`。
2. **填写上传**：填入您的姓名昵称、要留言的一句话、并选好手机里的照片原图，点击上传。
3. **后端拦截与裁剪**：
   - 后端会全自动对上传图片执行智能缩放，裁剪至实机屏幕的 `480x800`，压缩转码为 `.rgb565`。
   - 数据安全存入 `upload.db` 数据库。
4. **与 ESP32-S3 实机完美同频的高阶动效联动**：
   - ESP32 实机以 **30秒一次** 的频率自动向 `/api/upload/check` 接口轮询。
   - 当检测到有新上传照片时，实机自动高速下载该二进制数据并存入备用 PSRAM 缓冲区。
   - 随后，实机屏幕上会瞬间爆发出由我们编写的**电影级动效**：
     * **全屏半透明磨砂遮罩**（400ms 柔和淡入）。
     * **复古纸张模态弹窗**（在**弹簧物理 Overshoot 阻尼曲线**下优雅回弹拉开）。
     * **圆形彩色粒子**呈放射状爆破飞舞。
     * 发件人与您的留言寄语以 150ms 阶梯时序优雅显现。
     * 3.8秒后，新旧照片通过 **Crossfade 双图层跨帧透明度淡出** 实现平滑过渡！
